# PWA in FIREBALL CMS

PWA lets a site work as a regular browser site and as an installed app after it is added to the device home screen.

## Enabling

Open **Settings → PWA**:

- enable PWA;
- enable Push if needed;
- set the app name, short name, description and colors;
- choose a logo and startup image if they differ from the favicon;
- generate VAPID keys for Push.

The manifest and service worker are generated by the CMS automatically. Themes do not need extra files or scripts.

## App Icon

The manifest chooses the icon in this order:

1. site favicon;
2. PWA logo;
3. default FIREBALL CMS icon.

The CMS generates 48, 72, 96, 128, 144, 152, 192, 384 and 512 pixel icons, plus a maskable Android icon. Changing the favicon or logo refreshes the cache version.

## App Installation

On Android Chrome and compatible browsers you can use:

```html
<button data-pwa-install>Install app</button>
```

iPhone Safari does not support `beforeinstallprompt`. The user must use **Share → Add to Home Screen**. The CMS detects iOS Safari and shows the hint only when the site is not already opened as an app.

When opened as an app, the CMS adds:

```css
body.pwa
body.standalone
html.pwa
html.standalone
```

Themes can use these classes for safe-area handling, bottom bars and hiding install controls.

## Push Notifications

The CMS uses Web Push + VAPID without third-party services. Notification permission is never requested automatically. The user must click:

```html
<button data-pwa-enable-push>Enable notifications</button>
```

To disable:

```html
<button data-pwa-disable-push>Disable notifications</button>
```

On iPhone, Push works only for the site installed to the home screen and opened from its icon.

## NotificationService

Any module or plugin can send a notification:

```php
use App\Services\NotificationService;

NotificationService::sendToUser($userId, [
    'title' => 'New request',
    'body' => 'A new request was submitted.',
    'url' => base_href('/admin/contact-requests'),
    'tag' => 'contact-request',
]);
```

Supported fields: `title`, `body`, `icon`, `badge`, `image`, `url`, `tag`, `data`, `vibrate`, `timestamp` and `actions`.

## Offline

Full offline mode is not enabled yet. The service worker shows the **No connection** page only for regular navigation when the network is unavailable.

Admin, auth, profile, cart, orders, chat, notifications and private API requests are not cached.
