# PWA в FIREBALL CMS

PWA позволяет сайту работать как обычный сайт в браузере и как установленное приложение после добавления на главный экран устройства.

## Как включить

Откройте **Настройки → PWA**:

- включите PWA;
- при необходимости включите Push;
- задайте название приложения, короткое название, описание и цвета;
- укажите логотип и заставку, если они отличаются от favicon;
- сгенерируйте VAPID-ключи для Push.

Manifest и Service Worker создаются CMS автоматически. Темам не нужно добавлять отдельные файлы или скрипты.

## Иконка приложения

Manifest выбирает иконку в таком порядке:

1. favicon сайта;
2. логотип из настроек PWA;
3. стандартная иконка FIREBALL CMS.

CMS автоматически создаёт размеры 48, 72, 96, 128, 144, 152, 192, 384 и 512 пикселей, а также maskable icon для Android. При смене favicon или логотипа версия кэша обновляется.

## Установка приложения

На Android Chrome и совместимых браузерах можно использовать кнопку с атрибутом:

```html
<button data-pwa-install>Установить приложение</button>
```

На iPhone Safari нет события `beforeinstallprompt`. Пользователь должен открыть меню **Поделиться → На экран Домой**. CMS определяет iOS Safari и показывает подсказку только если сайт ещё не открыт как приложение.

Когда сайт открыт как приложение, CMS добавляет классы:

```css
body.pwa
body.standalone
html.pwa
html.standalone
```

Темы могут использовать их для safe-area, нижних панелей и скрытия кнопки установки.

## Push-уведомления

CMS использует Web Push + VAPID без сторонних сервисов. Разрешение на уведомления не запрашивается автоматически. Пользователь должен нажать кнопку:

```html
<button data-pwa-enable-push>Включить уведомления</button>
```

Для отключения:

```html
<button data-pwa-disable-push>Отключить уведомления</button>
```

На iPhone Push работает только для сайта, добавленного на главный экран и открытого с иконки.

## NotificationService

Любой модуль или плагин может отправить уведомление:

```php
use App\Services\NotificationService;

NotificationService::sendToUser($userId, [
    'title' => 'Новая заявка',
    'body' => 'Поступила новая заявка с сайта.',
    'url' => base_href('/admin/contact-requests'),
    'tag' => 'contact-request',
]);
```

Поддерживаются поля `title`, `body`, `icon`, `badge`, `image`, `url`, `tag`, `data`, `vibrate`, `timestamp` и `actions`.

## Offline

Полный offline-режим пока не включён. Service Worker отдаёт страницу **Нет подключения** только для обычной навигации, если сеть недоступна.

Не кэшируются админка, авторизация, профиль, корзина, заказы, чат, уведомления и API с приватными данными.
