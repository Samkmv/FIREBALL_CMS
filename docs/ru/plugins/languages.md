# Языковые файлы плагинов

Плагин должен хранить собственные переводы внутри своей директории и не добавлять строки в основные языковые файлы CMS.

## Структура

```text
plugins/
  toy-car-rental/
    lang/
      ru.php
      en.php
      de.php
      zh-cn.php
```

Файл языка возвращает обычный PHP-массив:

```php
<?php

return [
    'toy_rental_menu' => 'Прокат машинок',
    'toy_rental_start_ride' => 'Начать поездку',
];
```

## Загрузка

`PluginManager` автоматически регистрирует директорию `lang` активного плагина. Переводы неактивного плагина не подключаются.

CMS сначала ищет файл текущего языка сайта, например `lang/ru.php`. Если файла нет, используется fallback `ru.php`, затем `en.php`. Отсутствующий файл не должен ломать сайт.

## Ключи

Используйте уникальный префикс плагина:

```php
FireballPluginToyCarRental::t('toy_rental_start_ride');
```

Не используйте слишком общие ключи вроде `title`, `save`, `status`, чтобы не конфликтовать с ядром или другими плагинами.

## Безопасность

Переводы плагина подключаются только из директории самого плагина. При повреждённом языковом файле ошибка логируется, а CMS продолжает работать без падения.
