# theme.json

Файл `theme.json` описывает тему и используется CMS для списка тем, активации, импорта и экспорта.

## Пример

```json
{
  "name": "My Theme",
  "slug": "my_theme",
  "version": "1.0.0",
  "author": "Author Name",
  "description": "Custom FIREBALL CMS theme",
  "preview": "preview.png",
  "minCmsVersion": "1.5.0"
}
```

## Поля

- `name` — человекочитаемое название темы.
- `slug` — системное имя папки темы.
- `version` — версия темы.
- `author` — автор.
- `description` — описание для админки.
- `preview` — имя файла превью в корне темы.
- `minCmsVersion` — минимальная версия FIREBALL CMS.

## Правила slug

Slug должен содержать только:

- маленькие латинские буквы;
- цифры;
- дефис;
- подчёркивание.

Нельзя использовать пробелы, `../`, абсолютные пути и системные имена вроде `default`, `admin`, `core`, `public`, `vendor`.

## Совместимость

При импорте CMS проверяет `minCmsVersion`. Если тема требует более новую версию CMS, установка блокируется.

## Частые ошибки

- Slug в `theme.json` не совпадает с именем папки.
- JSON повреждён запятой в конце объекта.
- Preview указан как путь `../preview.png` вместо имени файла.
