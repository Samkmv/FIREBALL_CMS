# Редактор тем

Редактор тем доступен в административной панели: **Внешний вид → Редактор тем**.
Он работает только с файлами выбранной темы и не предоставляет доступ к ядру CMS.

## Открытие файлов

Выберите тему в верхней панели, затем файл в дереве. Поддерживаются PHP, HTML,
CSS, JavaScript, JSON, Markdown, TXT и SVG. Для изображений отображается
предпросмотр и форма замены.

## Создание файлов и папок

Кнопки **Создать файл** и **Создать папку** работают только внутри `templates`,
`partials`, `assets/css`, `assets/js` и `assets/images`. Имя не должно содержать
путь, скрытые сегменты или запрещённое расширение.

PHP-файлы разрешены только в `templates` и `partials`. Максимальный размер
текстового файла составляет 1 МБ, изображения — 5 МБ.

## Сохранение и проверка

Перед записью PHP CMS запускает синтаксическую проверку. Файл с ошибкой не
сохраняется. Для JSON проверяется корректность структуры, а для `theme.json`
также обязательные поля `name`, `slug`, `version`, `author`, `description` и
`preview`.

При изменении текста редактор предупреждает о несохранённых изменениях.

## Резервные копии

Перед сохранением, удалением и восстановлением CMS создаёт резервную копию в
`storage/theme-backups`. Для каждого файла сохраняются 20 последних версий с
датой, пользователем и размером.

Откройте **Резервные копии**, выберите версию и нажмите **Восстановить**.
Текущая версия также будет сохранена перед восстановлением.

## Защита default-темы

При открытии `default` отображается предупреждение и кнопка создания копии.
Файлы `theme.json` и `templates/layout.php`, а также системные корневые папки
нельзя удалять или переименовывать.

## Ограничения безопасности

- Все существующие пути проверяются через `realpath()`.
- Запрещены `../`, абсолютные пути, скрытые файлы и символические ссылки.
- Нельзя перейти в другую тему или открыть файлы ядра.
- Запрещены `.env`, `.htaccess`, `.user.ini`, Composer/NPM manifests, SQL,
  PHAR, исполняемые и shell-файлы.
- Действия и ошибки записываются в `storage/logs/theme-editor.log`.

Тема должна использовать Theme API и не должна выполнять SQL-запросы напрямую.
