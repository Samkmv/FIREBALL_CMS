# Редактор тем

Статус: **Beta**. Редактор тем является встроенным инструментом для безопасной
правки файлов темы и подготовки темы к разработке. Он доступен в админ-панели:
**Внешний вид -> Редактор тем**.

Редактор не изменяет Theme API и работает только с файлами выбранной темы.
Доступ к ядру CMS, другим темам и внешним путям запрещён.

## Возможности

- открытие файлов из дерева темы;
- создание файлов и папок внутри разрешённых директорий;
- переименование и удаление элементов;
- сохранение текстовых файлов;
- предпросмотр и замена изображений из `assets/images`;
- история резервных копий;
- восстановление любой сохранённой версии файла;
- предупреждение при редактировании системной темы `default`;
- создание копии системной темы перед изменениями.

## Рабочая область

Текущая версия использует обычный `<textarea class="theme-editor-code">`.
JavaScript редактора построен через adapter-слой, поэтому последующая замена на
Monaco Editor или CodeMirror не должна требовать переписывания открытия,
сохранения, проверки и восстановления файлов.

## Файлы и папки

Редактируемые текстовые форматы: PHP, HTML, CSS, JavaScript, JSON, Markdown,
TXT и SVG. Существующие изображения в `assets/images` можно просматривать и
заменять файлом с тем же расширением.

Новые файлы можно создавать только с расширениями `.php`, `.css`, `.js`,
`.json`, `.md` и `.txt`. PHP-файлы разрешены только в `templates` и `partials`.
Папки можно создавать внутри `templates`, `partials` и `assets`.

Ограничения размера:

- текстовые файлы: до 1 МБ;
- изображения: до 5 МБ.

## Проверка перед сохранением

Перед записью CMS выполняет проверки:

- PHP проверяется через `php -l`;
- при ошибке файл не сохраняется, а пользователь видит строку и описание ошибки;
- JSON должен быть валидным;
- `theme.json` дополнительно должен содержать `name`, `slug`, `version`,
  `author`, `description` и `preview`;
- `slug` в `theme.json` должен совпадать с папкой темы;
- SVG проверяется на небезопасные элементы и обработчики.

## Резервные копии и восстановление

Перед каждым сохранением, удалением и восстановлением создаётся резервная копия
в `storage/theme-backups`. Для каждой версии сохраняются:

- дата;
- пользователь;
- путь файла;
- размер.

Для одного файла хранится до 20 последних версий. Восстановление доступно из
панели **Резервные копии**. Перед восстановлением текущая версия файла также
сохраняется в историю.

## Безопасность

Редактор использует sandbox внутри текущей темы:

- все существующие пути проверяются через `realpath()`;
- запрещены `../`, абсолютные пути и скрытые сегменты;
- запрещены символические ссылки;
- запрещён доступ к ядру CMS;
- запрещён доступ к другим темам;
- запрещены `.env`, `.htaccess`, `.user.ini`, Composer/NPM manifests, SQL,
  PHAR, исполняемые и shell-файлы;
- операции восстановления сверяют metadata резервной копии с текущей темой и
  файлом.

Все действия и ошибки записываются в `storage/logs/theme-editor.log`: открытие,
сохранение, создание, удаление, переименование, восстановление, ошибки
синтаксиса и ошибки операций.

## Системная тема Default

При редактировании `default` CMS показывает предупреждение:

> Вы редактируете системную тему. Рекомендуется создать копию темы перед
> внесением изменений.

Используйте кнопку **Создать копию темы** и работайте с копией, если изменения
не являются частью разработки системного шаблона.

## Ограничения Beta

- редактор не является полноценной IDE;
- нет автодополнения Theme API;
- нет визуального diff между версиями;
- нет child themes. Этот этап является основой для следующего этапа разработки:
  **Child Themes**.
