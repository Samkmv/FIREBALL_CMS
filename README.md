# FIREBALL CMS

FIREBALL CMS - легкая PHP CMS с MVC-архитектурой, мультиязычностью, системой тем, блочным редактором, встроенным установщиком, аналитикой и центром обновлений.

Проект подходит для блогов, информационных порталов, корпоративных сайтов, каталогов, баз знаний и собственных веб-проектов.

---

### Аналитика

Встроенная аналитика включает:

* Трекинг посещений через `/api/analytics/track`
* Просмотры страниц
* Просмотры записей
* Статистику посещений
* Источники переходов
* Географию посетителей при включенном GeoIP
* Обновление и сброс данных из админ-панели

GeoIP является необязательным дополнением. Без MaxMind DB Reader или базы `.mmdb` визиты продолжают записываться без определения страны и города.

GeoIP можно отключить в `config/config.local.php`:

```php
'ANALYTICS_SETTINGS' => [
    'geoip_enabled' => false,
],
```

## Установка

1. Скопируйте проект на сервер или в локальное окружение.
2. Убедитесь, что корнем сайта является `public/` или что серверные rewrite-правила направляют запросы в `public/index.php`.
3. Установите зависимости, если папка `vendor/` отсутствует:

```bash
composer install
```

4. Откройте сайт в браузере.

Если CMS еще не установлена, автоматически откроется мастер установки:

```text
/install
```

Мастер установки позволяет:

* Выбрать язык
* Настроить подключение к базе данных
* Указать название сайта
* Настроить URL сайта
* Выбрать часовой пояс
* Создать первого пользователя Creator

После завершения установки система:

* Создает таблицы базы данных
* Создает необходимые системные записи
* Создает локальный конфигурационный файл
* Создает первого пользователя Creator
* Блокирует повторную установку

## Обновления

FIREBALL CMS поддерживает централизованное обновление через GitHub Releases и административную панель.

Возможности центра обновлений:

* Проверка новых версий
* Обновление из админ-панели
* Автоматический backup перед обновлением
* Выполнение миграций после обновления
* Rollback обновления
* Защита пользовательских файлов

Файл `update.json` описывает текущую версию, минимальную поддерживаемую версию, наличие миграций, обновляемые пути и защищенные директории.

Во время обновления защищаются:

* `config/config.local.php`
* `storage`
* `uploads`
* `public/uploads`
* `themes/custom`

## Структура проекта

```text
app/                 Контроллеры, модели, сервисы, компоненты и языки приложения
config/              Конфигурация, маршруты, версия и системные миграции
core/                Ядро CMS: роутер, MVC, сессии, БД, темы, кеш, middleware
database/            Схема, seed/demo-данные и миграции
docs/                Технические отчеты и документация проекта
helpers/             Общие helper-функции
public/              Публичная точка входа, assets и публичные локализации
storage/             Backup, GeoIP, логи и резервные копии тем
themes/              Темы оформления
tmp/                 Временные файлы
vendor/              Composer-зависимости
```

## Основные маршруты

Публичные:

* `/` - главная страница
* `/posts` - список записей
* `/posts/{slug}` - запись
* `/category/{slug}` - категория
* `/archive` - архив
* `/search` - поиск
* `/support` - поддержка
* `/support/articles` - статьи базы знаний
* `/contacts` - контакты
* `/login`, `/register`, `/forgot-password`, `/reset-password` - auth-сценарии
* `/two-factor-challenge`, `/two-factor-recovery` - 2FA-сценарии

Административные:

* `/admin` - панель управления
* `/admin/posts` - записи
* `/admin/pages` - страницы
* `/admin/categories` - категории
* `/admin/users` - пользователи
* `/admin/roles` - роли
* `/admin/settings` - настройки
* `/admin/settings/privacy` - приватность
* `/admin/settings/mail` - почта
* `/admin/security/logs` - журнал безопасности
* `/admin/analytics` - аналитика
* `/admin/files` - файловый менеджер
* `/admin/themes` - темы
* `/admin/theme-editor/{slug}` - редактор темы
* `/admin/updates` - обновления
* `/admin/system/database-maintenance` - обслуживание базы данных
